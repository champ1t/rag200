# บทที่ 4 ผลการดำเนินงาน

ระบบผู้ช่วยอัจฉริยะสำหรับงานเทคนิคและค้นหาข้อมูลองค์กร (Technical Knowledge Hybrid RAG System) ได้รับการทดสอบตามขอบเขตความสามารถที่กำหนดไว้ในสถาปัตยกรรมใหม่ (Hybrid Architecture) โดยแบ่งผลการทดสอบออกเป็น 4 ส่วนหลักเพื่อให้สอดคล้องกับกลไกการทำงานจริงของระบบ ได้แก่ การรักษาความปลอดภัยและจำแนกเจตนา, การจัดการบริบทสนทนา, ระบบค้นหาข้อมูลแบบกำหนดแน่ชัด และระบบประมวลผลความรู้ด้วย RAG ดังมีรายละเอียดผลการดำเนินงานดังต่อไปนี้

---

### 4.1 ผลการทดสอบกลไกป้องกัน และการจำแนกเจตนา (Governance Guardrails & Intent Routing Results)
ส่วนนี้คือประตูด่านแรกของระบบที่ทำหน้าที่คัดกรองความปลอดภัยและแยกแยะประเภทคำถาม (Classification) ก่อนส่งไปค้นหาข้อมูล:
*   **ผลการป้องกันคำถามนอกขอบเขต (Out-of-Scope Bypass):** ระบบสามารถตรวจจับและบล็อกคำถามที่ไม่เกี่ยวกับงานเทคนิคขององค์กร (เช่น คำถามทั่วไป, คำถามทางการเมือง หรือการเปรียบเทียบเชิงความเห็นส่วนตัว) ได้อย่างแม่นยำ โดยระบบจะตอบกลับด้วยข้อความปฏิเสธอย่างสุภาพทันที ทำให้ไม่สิ้นเปลืองทรัพยากรของ LLM
*   **ผลการตรวจสอบอุปกรณ์ (Vendor Scope Check):** ระบบสามารถดักจับชื่อผู้ผลิต (เช่น Huawei, ZTE, Cisco) ในข้อความคำถาม และตรวจสอบสิทธิ์การเข้าถึงข้อมูลของแบรนด์นั้นๆ ได้ หากเป็นแบรนด์ที่ไม่มีในระบบ SMC ระบบจะลดความสำคัญของการค้นหาและแจ้งผู้ใช้ทันที
*   **การจำแนกเจตนาแบบผสม (Hybrid Routing):** ระบบสามารถแยกแยะความแตกต่างระหว่าง "การขอเบอร์โทรคอลเซ็นเตอร์" (CONTACT_LOOKUP) และ "วิธีตั้งค่าเราเตอร์" (HOWTO_PROCEDURE) ได้อย่างถูกต้องผ่านการทำงานร่วมกันระหว่าง Keyword Matching และ LLM Prompting

### 4.2 ผลการทดสอบระบบจัดการบริบทและการสนทนาแบบต่อเนื่อง (Context Management & Pending State Results)
จุดเด่นที่พัฒนาเพิ่มเติมจาก RAG รูปแบบเดิม คือความสามารถในการจดจำและสานต่อบทสนทนา (Multi-turn Interaction):
*   **การจัดการสถานะค้างค้าง (Pending State Resolution):** เมื่อระบบค้นพบข้อมูลที่ต้องให้ผู้ใช้เลือก (เช่น เจอเบอร์โทรศัพท์ 3 แผนกที่ชื่อคล้ายกัน) ระบบสามารถหยุดรอคำตอบ (เช่น "กด 1, 2, หรือ 3") และเมื่อผู้ใช้พิมพ์เพียงตัวเลข ระบบสามารถนำตัวเลขนั้นไปจับคู่กับบริบทเดิมและดึงข้อมูลที่ถูกต้องออกมาได้ทันที (Numeric Selection)
*   **การเติมเต็มบริบท (Context Enrichment):** หากผู้ใช้ถามคำถามต่อเนื่อง เช่น "ศูนย์หาดใหญ่อยู่ไหน" ตามด้วยคำถามสั้นๆ ว่า "แล้วเบอร์โทรล่ะ" ระบบสามารถดึงคำว่า "ศูนย์หาดใหญ่" จากรอบที่แล้วมาต่อท้ายคำถามกลายเป็น "เบอร์โทรของศูนย์หาดใหญ่" ทำให้ตอบคำถามได้อย่างต่อเนื่องแบบไม่ต้องพิมพ์ใหม่

### 4.3 ผลการทดสอบระบบค้นหาข้อมูลแบบกำหนดแน่ชัด (Deterministic Lookup Results)
การทดสอบกระบวนการสืบค้นที่**ไม่ใช้ LLM สังเคราะห์คำตอบ** แต่เน้นความถูกต้องแบบ 100% (High Precision):
*   **การค้นหาบุคคลและทีมงาน (Directory Handler):** ระบบสามารถอ่านคำถามที่สะกดผิดเล็กน้อย หรือใช้คำย่อภาษาไทย (Colloquial words) แล้วแปลงเป็นชื่อทีมดั้งเดิมเพื่อค้นหาหมายเลขโทรศัพท์ หรือรายละเอียดของบุคคลในโครงสร้างองค์กรได้อย่างแม่นยำ ไม่เกิดปัญหา Hallucination (การมโนคำตอบของ AI)
*   **การค้นหาบทความแบบชี้เป้า (Title & Slug Exact Match):** หากคำถามตรงกับหัวข้อบทความในระบบ เช่น "คู่มือการตั้งค่า VPN" ระบบจะลัดคิวการค้นหา (Bypass Vector DB) และดึงลิงก์พร้อมข้อความสรุปส่งให้ผู้ใช้ทันที ทำให้ได้ความเร็วระดับ O(1)

### 4.4 ผลการทดสอบระบบประมวลผลความรู้ (RAG Pipeline Results)
การทดสอบด่านสุดท้ายสำหรับข้อมูลเชิงลึกทางเทคนิคที่ไม่มีโครงสร้างชัดเจน (Unstructured Data) เช่น คู่มืออุปกรณ์ขนาดยาว:
*   **การดึงข้อมูลตามความหมาย (Semantic Search):** ระบบสามารถใช้ Vector Database (ChromaDB) เพื่อสกัดและดึงเฉพาะ "ย่อหน้า" (Chunks) ที่เกี่ยวข้องกับอาการเสียที่พนักงานแจ้ง มาใช้เป็น Context ได้อย่างตรงประเด็น 
*   **ความถูกต้องของการสรุปความ (Grounding / No Hallucination):** ระบบ RAG สามารถสรุปเนื้อหาขั้นตอนการปฏิบัติงานจากเอกสารออกเป็นข้อๆ ได้อย่างชัดเจน โดยคงคำศัพท์เทคนิคไว้ตามต้นฉบับ **พร้อมแนบแหล่งอ้างอิง (Reference Link) กลับไปยังหน้าเอกสารของระบบต้นทางเสมอ**
*   **การสกัดและปกป้องคำสั่ง (Command Protection):** สำหรับเอกสารที่มีการระบุคำสั่ง (CLI Commands) ของอุปกรณ์เครือข่าย เช่น Huawei หรือ ZTE ระบบมีกลไกป้องกันคำสั่งบิดเบือน เพื่อให้พนักงานสามารถนำชุดคำสั่งนั้นไป คัดลอก-วาง (Copy-Paste) ในอุปกรณ์จริงได้โดยไม่เกิดข้อผิดพลาดด้าน Syntax ដែលเกิดจากการสร้างคำตอบใหม่ของ LLM
