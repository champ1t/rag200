# 2.2) ทฤษฎีและเทคโนโลยีที่เกี่ยวข้อง (Related Theories & Technologies)

ในโครงงานนี้มีการประยุกต์ใช้เทคโนโลยีปัญญาประดิษฐ์ สถาปัตยกรรมข้อมูล และกลยุทธ์การตัดสินใจขั้นสูง (Advanced Decision-Making) เพื่อพัฒนาระบบ Hybrid Retrieval-Augmented Generation (Hybrid RAG) ซึ่งมีความซับซ้อนและแม่นยำกว่าระบบ RAG ทั่วไป ทฤษฎีที่เกี่ยวข้องมีดังนี้:

### 2.2.1) โมเดลภาษาขนาดใหญ่ (Large Language Model - LLM)
LLM คือปัญญาประดิษฐ์ที่ถูกฝึกฝนด้วยข้อมูลข้อความมหาศาล จนมีความสามารถในการอ่าน สรุปความ และสร้างข้อความที่ซับซ้อนได้อย่างเป็นธรรมชาติ
*   **Role in Project:** ทำหน้าที่เป็น "สมอง" ในสองส่วนหลักคือ 
    1. **Intent Classification** (ใช้งานร่วมกับ Rule-based) เพื่อตีความความต้องการของผู้ใช้อย่างลึกซึ้ง
    2. **Generation** ในการนำข้อมูลที่ระบบค้นหามาได้ (Retrieved Context) มารวบรวมและเรียบเรียงเป็นคำตอบที่ลื่นไหล เป็นธรรมชาติ และตรงคำถามที่สุด
*   **Technology Used:** โมเดลตระกูล Llama3.2 (3B) ทำงานร่วมกับกรอบการทำงาน Ollama ซึ่งเป็น Open-source LLM ที่อนุญาตให้รันโมเดลภายในเซิร์ฟเวอร์ขององค์กร (On-premise) ทำให้รับประกันความปลอดภัยของข้อมูลที่เป็นความลับ (Data Privacy) ได้ 100%

### 2.2.2) สถาปัตยกรรมการค้นหาและสร้างคำตอบแบบผสมผสาน (Hybrid Retrieval-Augmented Generation)
RAG แบบดั้งเดิม (Vanilla RAG) อาศัยเพียงความคล้ายคลึงของเวกเตอร์ ซึ่งมักพบปัญหาการดึงข้อมูลผิดพลาด (Hallucination หรือ Low Precision) โครงงานนี้จึงพัฒนาเป็น **Hybrid RAG** ที่แยกแยะรูปแบบคำถามก่อนตัดสินใจค้นหา:
*   **Deterministic Retrieval:** หากคำถามเป็นเรื่องตายตัว เช่น เบอร์โทรศัพท์, โครงสร้างทีม, หรือชื่อบุคคล (Contact/Team/Person Lookup) ระบบจะใช้การค้นหาแบบระบุตัวตน (Keyword/Exact Match) จาก Directory โดยตรง เพื่อความแม่นยำ 100% โดยไม่ต้องพึ่งหา LLM
*   **Semantic Retrieval:** หากคำถามมีความซับซ้อน เช่น วิธีการตั้งค่า หรือคู่มือแก้ปัญหา ระบบจึงจะค้นหาความหมายผ่าน Vector Database เพื่อความยืดหยุ่นทางภาษา
*   **Benefit:** ช่วยให้ประหยัดทรัพยากรประมวลผล (Compute Power) ลดระยะเวลาการรอคำตอบ (Latency) และเพิ่มความแม่นยำ (High Precision) ได้เทียบเท่าระบบฐานข้อมูลดั้งเดิมแต่ยังคงใช้งานผ่านภาษาธรรมชาติได้

### 2.2.3) การสร้างเวกเตอร์แทนความหมาย (Vector Embedding) และ ฐานข้อมูลเวกเตอร์ (Vector Database)
กระบวนการวิเคราะห์และการจัดเก็บข้อมูลเพื่อรองรับ Semantic Search
*   **Vector Embedding:** การแปลงข้อมูล "ข้อความ" (Text) ให้กลายเป็น "ตัวเลขทางคณิตศาสตร์" ในรูปแบบหลายมิติ (Multi-dimensional Vector) หากข้อความใดมีความหมายหรือบริบทใกล้เคียงกัน เวกเตอร์จะชี้ไปในทิศทางเดียวกันในสเปซทางคณิตศาสตร์ (Cosine Similarity สูง)
*   **Vector Database (ChromaDB):** ทำหน้าที่เป็นคลังสมองของระบบ ที่รองรับการค้นหาจุดเวกเตอร์ที่ใกล้เคียงที่สุด (Similarity Search) ได้อย่างรวดเร็วระดับมิลลิวินาที แม้ผู้ใช้จะใช้คำศัพท์คนละคำกับในเอกสาร แต่หากสื่อความหมายเดียวกัน ระบบก็สามารถดึงข้อมูลที่ถูกต้องออกมาได้

### 2.2.4) การจัดการบริบทของการสนทนา (Conversational Memory & Context Management)
การที่ระบบสามารถสนทนาต่อเนื่องได้อย่างมนุษย์ (Multi-turn Conversation)
*   **Short-Term Memory (Pending State):** กลไกการจดจำสถานะที่ค้างอยู่ เช่น หากระบบค้นพบคำตอบหลายทางเลือก และถามผู้ใช้กลับว่า "ต้องการข้อมูลของบุคคลใด (กด 1 หรือ 2)" ระบบจะจำบริบทนี้ไว้เพื่อประมวลผลคำตอบถัดไปที่เป็นตัวเลขให้ถูกต้อง
*   **Context Enrichment:** เมื่อผู้ใช้ถามคำถามต่อเนื่อง (Follow-up Query) ระบบจะดึง "เจตนา" (Intent) และ "คำนาม" (Entity) จากประโยคก่อนหน้า มาเติมเต็มในประโยคปัจจุบัน เช่น หากถาม "ศูนย์หาดใหญ่" แล้วถามต่อว่า "เบอร์อะไร" ระบบจะเข้าใจว่าหมายถึง "เบอร์ของศูนย์หาดใหญ่" โดยอัตโนมัติ

### 2.2.5) การประมวลผลภาษาธรรมชาติและกลไกรักษาความปลอดภัย (NLP Pre-processing & Governance Guardrails)
เพื่อให้ระบบมีความเสถียรและปลอดภัยในการใช้งานระดับองค์กร (Enterprise-grade)
*   **Colloquial Noise Removal:** การตัดคำรบกวนหรือคำสร้อยในภาษาไทย (เช่น "คืออะไรหว่า", "รบกวนขอ") เพื่อให้เหลือเพียงเนื้อหาที่สำคัญ (Core Intent) ก่อนนำไปค้นหา
*   **Entity Detection & Vendor Normalization:** ตรวจจับคำเฉพาะ (เช่น ชื่อแบรนด์อุปกรณ์ Huawei, Cisco) และปรับรูปแบบให้เป็นมาตรฐานเดียวกันก่อนการประมวลผล
*   **Governance Guardrails (Out of Scope Bypass):** ชั้นกรองความปลอดภัยที่จะบล็อกคำถามที่ไม่เกี่ยวข้องกับคู่มือทางเทคนิคขององค์กร (เช่น กฎหมาย, ความปรารถนาส่วนตัว) ก่อนที่จะส่งให้ LLM ประมวลผล เพื่อลดการสิ้นเปลืองทรัพยากรและป้องกันโมเดลสร้างคำตอบที่ผิดปกติ
