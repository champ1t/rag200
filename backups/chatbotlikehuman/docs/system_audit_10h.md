# รายงานการตรวจสอบระบบเชิงลึก (10 ชั่วโมงล่าสุด) 🕵️‍♂️
**ฉบับตรวจสอบความถูกต้องหลังการ Freeze ระบบ**

รายงานนี้สรุปการปรับปรุงระบบ RAG ในส่วนของ Logic และการตอบคำถาม โดยเน้นการแก้ที่ **"ต้นเหตุ (Root Cause)"** เพื่อให้ระบบรองรับภาษาไทยที่เป็นธรรมชาติได้โดยไม่ต้องพึ่งพาการดัก Keyword แบบแมนนวล

---

## 1. การปรับปรุงระบบ Normalization (หัวใจหลักของปัญหา OLT/ONT)
**ไฟล์ที่แก้ไข**: `src/core/chat_engine.py` (คลาส `ProcessedCache`)

### ❌ ปัญหาเดิม (The "Ambiguity" Root Cause)
เมื่อผู้ใช้ถามว่า *"ผมจะกำหนดค่าบน OLT"* ระบบดั้งเดิมจะพยายามหาคำว่า "ผมจะ" และ "บน" ในชื่อบทความด้วย เมื่อหาไม่เจอ คะแนนความมั่นใจจะตกลงมาอยู่ที่ **0.9** 
- กฎความปลอดภัย (**Ambiguity Guard**) ระบุว่าถ้าคะแนนอยู่ที่ 0.9 และมีหลายคำตอบที่ใกล้เคียงกัน (เช่น คู่มือ ZTE OLT, C300 OLT) ให้ระบบ "หยุดตอบ" และถามผู้ใช้ให้เลือกแทน
- ผลคือ: พิมพ์ถูก (OLT) กลับโดนดักถามซ้ำ ส่วนพิมพ์ไม่ตรง (ONT) กลับผ่านไปใช้ระบบ Vector Search ที่ฉลาดกว่าได้

### ✅ การแก้ไขที่ต้นเหตุ (Thai-Semantic Stripping)
ผมได้อัปเกรดเครื่องมือ `normalize_for_matching` ให้ทำงานแบบ **Context-Aware** ดังนี้:
- **ลอกเศษภาษา (Noise Stripping)**: เพิ่ม Logic การลบคำสรรพนาม (ผม, หนู), คำบอกเจตนา (จะ, อยาก), คำเชื่อม (บน, ที่, ของ) และคำลงท้าย (ครับ, ค่ะ, หน่อย) ออกจากชุดที่ใช้ค้นหา "ก่อน" เริ่มการคำนวณคะแนน
- **ผลลัพธ์**: ประโยค *"ผมจะกำหนดค่าบน OLT"* จะถูกมองเห็นเป็นแค่ **`"กำหนดค่า OLT"`** 
- **Impact**: คะแนนความแม่นยำพุ่งขึ้นไปที่ **0.98** ทำให้ระบบ "มั่นใจ" ว่าเจอคู่มือที่ถูกต้องแน่นอน และข้ามการถามซ้ำ (Ambiguity) ไปส่งลิงก์ให้ผู้ใช้ทันที

## 2. การปรับปรุงระบบ Routing (DispatchMapper)
**ไฟล์ที่แก้ไข**: `src/rag/handlers/dispatch_mapper.py`

### ❌ ปัญหาเดิม
ระบบ Dispatcher (ตัวจัดการจังหวัด) ทำงาน "ขยัน" เกินไป โดยการดักจับคำว่า "วงจรเช่า" หรือ "จ่ายงาน" แล้วพยายามแยกส่วนข้อมูลตามจังหวัดเสมอ
- ถ้าผู้ใช้ถามหา "คู่มือกลาง" หรือ "วิธีทำ" โดยรวม ระบบจะติดแหง็กอยู่ที่การขอชื่อจังหวัด

### ✅ การแก้ไขที่ต้นเหตุ (How-To Bypass)
- **Logic**: เพิ่มเงื่อนไขว่า หากในคำถามมี Keyword ประเภท "วิธีการ", "ขั้นตอน", "คู่มือ" **และ** ไม่มีการระบุชื่อจังหวัด ให้ตัว Dispatcher "ถอยออกไป" (Yield)
- **ผลลัพธ์**: คำถามเชิงระเบียบปฏิบัติจะถูกส่งต่อไปยังสารานุกรม RAG หลัก เพื่อทำการสรุปเนื้อหา (Summarize) ทั้งบทความให้ผู้ใช้อ่านแทนการเจาะจงรายจังหวัด

## 3. การรองรับภาษาพูด (Colloquial Resilience)
**ไฟล์ที่แก้ไข**: `src/ai/safe_normalizer.py` และ `src/rag/handlers/contact_handler.py`

- **2-Stage Cleaning**: เพิ่มระบบ LLM-Rewriter เพื่อ "แปล" ภาษาแชท (เช่น *เบอร์คนชื่อ...ดิ๊*, *ขอของทีม...หน่อยนะจ๊ะ*) ให้กลายเป็น "คำค้นมาตรฐาน" ก่อนส่งให้ Database
- **Entity Recovery**: ทำให้ระบบจำแนกชื่ออุปกรณ์ทางเทคนิค (U-MUX, ONU, OLT) ได้แม่นยำขึ้น แม้จะถูกล้อมหน้าล้อมหลังด้วยประโยคพูดคุยยาวๆ
