หน้าตา Dslam

มีไฟบอกสถานะของ Port ทั้ง 48 Port ว่าใช้งานหรือไม่ใช้งาน


DSLAM (ชื่อเต็ม: Digital Subscriber Line Access Multiplexer) หรือที่มักเรียกกันว่า "ดีสแลม" ใช้ในการแยกสัญญานจากสายทองแดง จากส่วนที่ให้บริการสัญญานโทรศัพท์และอินเทอร์เน็ต (ส่วนใหญ่จะเป็นที่ให้บริการโทรศัพท์ เช่น ทีโอที) ซึ่ง DSLAM จะแยกสัญญานออกเป็นสองสัญญานคือ สัญญานโทรศัพท์ และสัญญานอินเทอร์เน็ต เพื่อให้หน่วยงานที่รับผิดชอบ เช่นสัญญานโทรศัพท์ DSLAM จะส่งสัญญานผ่าน ATM แล้วส่งไปให้องค์การโทรศัพท์ทำการเชื่อมต่อเลขหมายปลายทาง ส่วนสัญญานอินเทอร์เน็ต DSLAM ก็จะส่งไปย้งผู้ให้บริการอินเทอร์เน็ตผ่านเครื่องส่งสัญญานแบบ ATM ไปยังหน่วยงานบริการอินเทอร์เน็ต(ISP)เพื่อทำการเชื่อมต่ออินเทอร์เน็ตและให้บริการ

ATM ย่อมาจาก Asynchronous Transfer Mode ซึ่งเป็นหนึ่งในเทคโนโลยีของระบบ WAN ใช้กันทั่วไปในองค์กรที่บริการเกี่ยวกับการสื่อสาร

 

Node DSLAM ทำหน้าที่ในการรวม User แต่ละ User แล้วส่งออกในระดับ IMA Group ( การรวม E1 ตั้งแต่ 1 E1 เป็นต้น ไปเพื่อแชร์ Bandwidth ซึ่งจะกล่าวต่อไป ) หรือเป็นการเชื่อมต่อในระดับ STM-1 เพื่อส่งให้ระบบ ATM , BRAS ซึ่งแล้วแต่ผู้ให้บริการจะเลือกใช้ นอกจากนี้ Node DSLAM ยังเป็นตัวกำหนดความเร็วที่เราๆท่านๆขอไปกับผู้ให้บริการด้วยล่ะครับ ซึ่งการกำหนดนี้ผู้ให้บริการจะทำเป็น profile ไว้ เมื่อลูกค้ารายไหนต้องการความเร็วเท่าไรก็เรียก profile นั้นออกมาใช้ครับ ส่วนการกำหนดอัตราการรับส่งที่บอกว่า 1:20 หรือ 1: 50 นั้นก็อาจจะกำหนดจาก Traffic Table ที่ Node DSLAM แล้วดึงมาใช้ตอนกำหนด ค่า vpi vci ให้กับลูกค้าแต่ล่ะรายนะครับ แต่การกำหนดอัตรารับส่งบางครั้งที่สามารถกำหนดที่ ATM ก็ได้ครับ ตอนนี้เรามาดูกันว่าค่า VPI VCI ที่ผู้ให้บริการกำหนดให้เรา Set ในแต่ละรายนั้นเค้าให้เรา set ไปทำไม ซึ่งคำตอบสามารถดูได้จากรูปด้านล่างครับ ซึ่งค่า PVI VCI ที่ผู้ให้บริการกำหนดมานั้นผมได้สรุปไว้ตามตารางที่หัวข้อเนื้อหาสาระครับ ก็ลองเข้าไปดูนะครับ

ค่า vpi vci ตามรูปเป็นค่าที่เรา set ไว้ที่ modem ก็คือค่า 0 35 ตามลำดับ ส่วน 0/1/0 นั้นก็หมายความถึง Sheft /Card/Port 0 ของ Node ซึ่ง Caed 1 ก็คือ Line Card นั้นเอง ซึ่งที่ตัว Node DSLAM จะทำการกำหนดค่า VPI VCI นี้รอไว้ที่ตัว Node ครับ ดังนั้นถ้าพี่ๆเพื่อนๆท่านใดกำหนดผิดจากค่าที่ผู้ให้บริการกำหนด นั้นก็จะหมายความว่า ท่านหมดสิทธิ์เล่นเน็ตครับ....

เมื่อ Line card รับค่า vpi vci จาก Modem แล้วที่ Node DSLAM จะส่งค่า vpi vci ให้ออกทางที่ MMXC Card โดยกำหนดค่า vpi vci ขึ้นมาใหม่ และการกำหนด VPI ตรงส่วนนี้เป็นเป็นตัวกำหนดค่า ของ VP ของ Node ด้วย เนื่องมาจากที่ตัว BRAS นั้นมีการจำกัดจำนวนค่า VP ไว้ต่อพอร์ต( ถ้าจำไม่ผิดนะครับ 1 พอร์ตจะรองรับค่า VP ได้ประมาณ 254 VP ครับ ) และค่า VP ในพอร์ตจะต้องไม่ซ้ำกัน ดังนั้นจึงต้องกำหนดค่า VP ขึ้นมาใหม่ เพราะทุก Node รับค่า VPI จากลูกค้าเป็น VPI 0 ถ้าไม่เปลี่ยน BRAS ก็จะไม่ทราบว่า User นี่มาจาก Node ไหนและเวลจ่าย IP Address ก็จะจ่ายให้ใครครับ เพราะทุกตัวเข้ามาด้วย VP 0 หมด

รูปแสดงการเชื่อมต่อค่า VPI,VCI จาก User ที่ Node DSLAM


รูปแสดง IMA Group

 

อุปกรณ์ DSLAM

ตอนนี้เรามาดูรูป Node กันเลยดีกว่าครับ ซึ่งเป็นรูปที่เท่าที่จะหาได้นะครับ จากรูปด้านล่างจะเป็น Node DSLAM รุ่นใหญ่ครับ ซึ่งส่วนมากแล้วเจ้า Node นี้จะใช้รองรับลูกค้าได้มากที่สุดครับ เนื่องจากมี Slot ใช้งาน 0-15 slot ( แต่ส่วนมากจะไม่ค่อยนำมารองรับลูกค้าซักเท่าไหร่ เพราะอาจมากเกินความจำเป็น ซึ่ง Node รุ่นนี้ส่วนมากจะมีไว้สำหรับเชื่อมต่อ Node รุ่นที่ต่ำลงไปมากกว่าเพื่อรวบรวม User ทั้งหมดและส่งออกไปหา BRAS , STM-1, STM-4 แล้วแต่กรณี แต่ก็จะนำมารองรับ User เหมือนกันในกรณีที่ Node รุ่นกลางรองรับลูกค้าไม่ไหวครับ ) และ slot 7-8 นั้นเป็น slot ที่มีไว้สำหรับปลั๊กการ์ด MMXC ครับซึ่ง slot 7 จะเป็น work ส่วน slot 8 จะเป็น standby ครับส่วน slot ที่เหลือก็อาจจะปลั๊ก ADLE,SPLC, LANC ,FRC , AIU,G.SHDSL ครับ แต่ในที่นี้ผมจะกล่าวก็มีแค่ ADLE ( หรือ ADLDในรุ่นเล็ก ),MMXC ( ATM ในรุ่นเล็ก ),AIU ครับ

รูปแสดง Node DSLAM รุ่นใหญ่

ต่อไปเรามาดู Node ขนาดกลางกันบ้างนะครับ (รูปด้านล่าง ) เจ้า Node นี้แตกต่างจาก Node รุ่นใหญ่แน่นอนครับที่เห็นๆกันเลยก็คือรูปร่างของ Node ซึ่งมันจะมีขนาดเล็กกว่า slot ก็น้อยกว่ากล่าวคือมีแค่ 1-7 slot เท่านั้นเอง แต่ที่เห็นๆอีกอย่างคือ ตำแหน่งปลั๊กการ์ด MMXC นั้นมีอยู่แค่ Slot เดียวคือ slot 7นั้นก็หมายความว่าถ้าเจ้าการ์ดนี้เจ๊ง Node นี้ทั้ง Node ก็เจ๊งทั้ง Node หรือถ้ามีการต่อพ่วงตัวลูก ตัวลูกมันก็จะเจ๊งไปด้วยล่ะครับ ซึ้งต่างจาก Node ขนาดใหญ่จะมี Slot 8 ไว้คอยทำงานแทนเมื่อ slot 7 เสียครับ และที่เห็นๆอีกอย่างก็คือการรองรับลูกค้าได้น้อยกว่ารุ่นใหญ่เนื่องจากมี slot น้อยกว่านั้นเอง แต่ที่เหมือนกันเลยก็คือ การ์ดของ Node ขนาดกลางและการ์ดของ Node ขนาดใหญ่สามารถใช้แทนกันได้เลยครับเนื่องจากมันเหมือนกันทุกประการเลยที่เดียวครับกระผม

รูปแสดง Node DSLAM รุ่นกลาง

DSLAM Card

ดังที่กล่าวไปแล้วนะครับว่าผมจะกล่าวถึงการ์ด 3 การ์ดคือ ADLE ( หรือ ADLDในรุ่นเล็ก ),MMXC ( ATM ในรุ่นเล็ก ),AIU เท่านั้นครับที่ใช้เกี่ยวกับการให้บริการอินเตอร์เน็ตของให้ผู้ให้บริการในขณะนี้


การ์ดแรกดังรูปด้านล่างเป็นการ์ด ADLE ซึ่งมี 32 พอร์ต ซึ่งหน้าที่ของการ์ด ADLE นั้นก็มีไว้สำหรับเขื่อมต่อกับลูกค้า ( ก็เราๆท่านๆนี่แหละครับ ) เข้ากับตัว Node DSLAM ครับ ซึ่งถ้าเป็น Node ขนาดใหญ่จะใช้ ADLE Card ส่วน Node ขนาดเล็กจะใช้ Card ADLD ซึ่งมี 16 port ต่อ 1card ครับ

รูปแสดง Line Card ( ADLE )

การ์ดต่อมาที่จะกล่าวคือ การ์ด MMXC ซึ่งจะว่าไปแล้วเจ้าตัวนี่เป็นพระเอกของ Node DSLAM เลยที่เดียวหรือเปรียบเทียบกับคอมพิวเตอร์ก็เป็น CPU เลยล่ะครับ เพราะว่าเจ้าการ์ด MMXC นี้จะเป็นตัวเก็บค่า Config ทั้งหมดใน Node ไว้ที่ตัวมันไม่ว่าการเก็บค่า VPI VCI ที่กำหนดให้ลูกค้า หรือกำหนด profile และ traffic table ดังที่กล่าวข้างต้น นอกจากนี้ตัวมันเองนอกจาก interface กับการ์ดอื่นๆใน Node แล้วมันก็ยังสามารถต่อเชื่อมกับ Node อื่นๆได้อีกซึ่งการเชื่อมต่อนั้นอาจเป็น แบบ STM-1 หรือ STM-1 ATM; STM-4 VPRing,IMA แล้วแต่ว่าจะเลือกใช้แบบไหน

รูปแสดง MMXC Card

การ์ดสุดท้ายเป็นการสำหรับเชื่อมต่อระดับ IMA ,STM-1ซึ่งผมเคยกล่าวดังเริ่มบทความแล้วนะครับว่า Node DSLAM รุ่นใหญ่นั้นส่วนมากจะใช้รองรับจำนวน User จาก Node รุ่นที่ต่ำกว่ามันลงมาและการเชื่อมต่อนั้นจะต้องใช้ card AIU นี่แหละครับในการเชื่อมต่อ จากนั้นก็ทำการกำหนดค่า PVP ให้มาที่ Card MMXC เพื่อส่งออกอีกครั้ง โดยการ์ด AIU นั้นจะมี Port ( E1) ทั้งหมด 8 E1 ( 0-7 )นอกจากนั้นยังสามารถเชื่อมต่อในระดับ STM-1 ได้อีกด้วยครับ ( อาจจะใช้เชื่อม Node DSLAM ขนาดใหญ่ เข้าด้วยกัน )
รูปแสดง AIU Card

รูปด้านล่างเป็นลักษณะการเชื่อมต่อ Node รุ่นต่างๆซึ่งจะเห็นได้ว่า Node DSLAM ขนาดเล็กนั้นไม่สามารถต่อตัวลูกพ่วงได้เลยครับส่วน Node DSLAM ขนาดกลางและใหญ่สามารถต่อตัวลูกพ่วงได้อย่าไรก็ตามการต่อใช้งานจริงๆอาจจะไม่ต่อแบบนี้ก็ได้แล้วแต่จะใช้งาน เช่น ต่อแบบไม่พ่วงตัวลูกเลยคือทุก Node เขื่อมต่อเข้ากับตัวลูกหมดโดยไม่ผ่าน Node อื่นซึ่งมีข้อดีคือ การต่อแบบพ่วงถ้า Node ต้นทางมีปัญหา ตัวลูกก็จะล่มไปด้วย แต่ถ้าต่อแบบต่อตรงปัญหาที่ส่วนนี้ก็จะหมดไป แต่ก็มีข้อเสียเหมือนกันคือทำให้ต้องติดตั้ง Node DSLAM ขนาดใหญ่เพิ่มเพื่อรองรับจำนวน Node ที่ต่อตรงเข้ามาครับ
